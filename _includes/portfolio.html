As a final project of the coding bootcamp I have worked in a team of 5 remote developers building a Ruby On Rails Chess Application. We have employed the following best practices:


  • TDD using gems such as RSpec and FactoryGirl.
  • CI and CD was implemented via CodeShip.
  • Trello was used to manage the project.
  • Version Control is done with Git and GitHub.
  • RuboCop gem makde sure our code complied with the Ruby Style Guide.

I have written the following logic:

  • Real-time database update of piece location on the board via AJAX. See the PR.
  • Check state of the game. PR.
  • Prevention logic for King so that it cannot move to cell in check as well as piece logic not to open their own king. PR.
  • Checkmate logic. PR.

While building this site, I have learned Jekyll and also got introduced to Bower and Gulp.
